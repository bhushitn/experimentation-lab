Experimentation Lab

A simulation engine where the true treatment effect is known by construction, used to break A/B tests the ways they break in production and measure the damage exactly.

Three layers: a tested Python engine that simulates experiment pathologies against known ground truth; a review agent that turns a design into a decision memo grounded in those simulations; and these explainers, where every number and every animation is generated by committed, tested code. Source on GitHub.

Explainers

  • Peeking: how checking your test daily manufactures false positives
    4,000 replays of the same null experiment: daily p-value checks reject 24% of the time at a nominal 5%, and an alpha-spending boundary restores it.
  • What randomization actually buys you
    Adopters versus everyone else reads 1.94; the truth is 0.50. Randomizing the same population recovers it.
  • Network effects: when your control group is contaminated by design
    User-level randomization on a social graph reads 0.52 against a true launch effect of 0.80; cluster randomization recovers it.
  • The subgroup fishing expedition
    Scanning 20 null segments finds a 'significant' subgroup in 63.8% of experiments.
  • Choosing the right test
    On a 70%-zeros revenue metric, Mann-Whitney detects a real spend lift 68% of the time; Welch's t, 100%.
  • CUPED and variance reduction
    A 28-day pre-period covariate removes 48% of the variance: the same power at roughly half the sample.
  • Power, MDE, and how long to run
    A 1-point lift on a 10% rate needs 14,000 users per arm; half the lift needs four times that.
  • From p-value to decision
    One experiment end to end: the novelty spike, the honest window, and the memo.
  • Sample ratio mismatch: the smoke alarm
    Assigned 50/50, logged 98,022 vs 100,000: invisible to the eye, decisive to the chi-square, and fatal to the readout.
  • Switchbacks: testing when the marketplace is the unit
    One-hour windows understate the effect by 39% through carryover; day-long windows double the variance. Burn-in resolves it.
  • When you cannot randomize
    Difference-in-differences recovers the truth under parallel trends and overstates it 2x when trends diverge; the placebo test tells you which world you are in.
All data on this site is synthetic, generated by the repository's simulation engine with fixed seeds. That is the point: because the true effect is set by construction, every bias and error rate shown here is measured against an answer key, which no real dataset allows. The running product examples describe a fictional short-video app.
