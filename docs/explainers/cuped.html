CUPED: the sample size you already paid for

Your users were generating data for weeks before the experiment started. CUPED spends that history to cancel noise you would otherwise buy with sample size.

Simulated experiment, 100,000 users, true effect 0.20, 28-day pre-period covariate. Reproduce this figure.

Users predict themselves

Plot each user's 28-day pre-period average against their outcome during the experiment. The cloud leans hard: heavy users stay heavy, light users stay light. That persistence is variance the experiment does not need to relitigate, because the pre-period is identical in both arms by randomization.

Subtract the predictable part

CUPED replaces each outcome y with y - theta (x - mean(x)), where x is the pre-period value and theta = 0.97 is the regression slope of the lean. Centering x keeps the treatment effect untouched; the subtraction removes only the part of y the pre-period already explained.

48% of the variance, gone

Measured variance reduction: 48.4%, matching the analytic value (the squared pre/post correlation, 48.3%) to a decimal. The confidence interval narrows from 0.105 to 0.075: 28% tighter, on the same users, with the same test.

Variance is sample size

A 48% variance cut means reaching the same power with roughly half the users, or halving the noise floor of every experiment on this metric permanently. The covariate can be anything pre-assignment that correlates with the outcome; the same metric measured pre-period is usually the best one. Deng, Xu, Kohavi and Walker introduced the method at WSDM 2013.

The memo you write instead

Recommendation

Adopt CUPED with the 28-day pre-period metric as the covariate for all experiments on this metric. Measured on a simulated experiment with known truth: 48.4% variance reduction (matching the analytic 48.3%), confidence intervals 28% narrower, equivalent to roughly doubling the traffic at zero cost.

Method note: Deng, Xu, Kohavi, Walker, "Improving the Sensitivity of Online Controlled Experiments by Utilizing Pre-Experiment Data", WSDM 2013. Implementation validated against the analytic variance reduction in tests/unit/test_stats.py.
