Network effects: when your control group is contaminated by design

A short-video app tests a share-to-friends prompt. Treated users send clips into their follow graph, so control users' feeds change too. The experiment is measuring a launch that leaks.

Simulated on a 20,000-user social graph (100 clusters), direct effect 0.5, spillover 0.3, so the global launch effect is 0.8. Reproduce this figure.

The number the decision needs

Launching means everyone gets the prompt. The launch effect is direct effect plus spillover: 0.8 in this simulation, set by construction. The dashed line is that target. The experiment's job is to estimate it.

Randomize users, get the wrong number

Under user-level randomization the estimator reads 0.52: the direct effect and essentially none of the spillover. This is not noise, and more sample will not fix it; the average of a bigger sample converges to the same wrong number.

Why: both arms have the same neighbors

With half of all users treated, a treated user's friends are about 50% treated, and so are a control user's. The spillover hits both arms equally and subtracts out of the contrast: the measured exposure gap between arms is 0.003. Control is not untreated; it is half-treated by design.

Randomize the graph, not the user

Assign whole friend-clusters to one arm and treated users live in mostly-treated neighborhoods (exposure gap 0.95). The same estimator now reads 0.83, within noise of the 0.8 target, degraded only by the few edges that cross clusters.

The price and the rule

Cluster randomization costs power: the effective sample size is closer to the number of clusters than the number of users, so size the experiment on clusters. The rule: if the feature moves anything between users (sharing, messaging, matching, marketplaces), decide the randomization unit before launch, not in the readout.

The memo you write instead

Recommendation

Randomize follow-graph clusters, not users, and size on the number of clusters. Under user-level randomization this experiment would read 0.52 against a true launch effect of 0.80, missing the spillover entirely (measured on a simulated graph where the truth is set by construction); cluster randomization recovers 0.83.
